# 履歴書スキル

履歴書が人間に読まれるのは約 7 秒、その前に ATS スキャンがある。だから*ざっと読める*こと、*実績主導*であること、*キーワードが揃っている*ことが必須——職務内容の再掲ではだめだ。このスキルはあなたの経歴を定量化された実績の箇条書きに変換し、1 カラム構成（ATS セーフ）で組み、ターゲット職種に合わせて調整する。**Paper** または **Modern** の PDF テーマで書き出せば、組版された仕上がりになる。

## ラフな素材から仕事をする

手元にあるのは走り書きのメモ、部分的な経歴、あるいはターゲット職種名だけ、ということがよくある。**それでも必ず、完成した、そのまま使える履歴書を納品する**——質問のために立ち止まらず、`[会社名]` や `[数値を追加]` のような角括弧のプレースホルダーも残さない。詳細が欠けている場合は、素材の残りとターゲット職種から具体的で現実的な内容を推定し、推定した箇所には*（推定——要確認）*と印を付けて、ユーザーが検証すべき箇所を分かるようにする。具体的でラベル付きの推定は、空欄や確認質問に常に勝る。

すべての実績を定量化する。**ユーザーの実数を最優先**——数字が与えられていればそのまま使う。指標が本当に欠けているときだけ、職務を成果志向の実績に書き換え（*結果*を書く。例：「モバイルアプリ v1 をリリースし、最初のユーザー群を獲得」）、**守りに入った控えめな見積もり**が可能な場合のみ数字を足し、*（推定——要確認）*と印を付ける。実在の人物の履歴書で数字を黙って捏造・誇張してはならない——印のない創作指標は、欠けた指標より悪い唯一のものだ。

出力は**完成した履歴書のみ**（と短い調整メモ）——前置きなし、「履歴書はこちらです」なし、自分のやったことへのメタ解説なし。

## 入力（未提供のものは推定する——推定にはラベルを）

- **ターゲット職種 / 求人票**——調整とキーワード整合のため（汎用履歴書は負ける）。
- **経歴**——役職、期間、やったこと/成し遂げたこと（ラフなメモで可。定量化はスキルの仕事）。
- **スキル、ツール、学歴、資格。**
- **シニアリティと形式の希望**——逆時系列（デフォルト）か職能型か。1 ページ（大半）か 2 ページか。

## 出力形式

1 カラム・ATS フレンドリーの履歴書を以下の順で：

### [氏名]
[ターゲット職種名] · [都市 / リモート] · [メール] · [電話] · [LinkedIn/ポートフォリオ]

**サマリー**——2〜3 行：あなたは何者か、最も強い証拠、狙っている方向。「結果志向のプロフェッショナル」のような埋め草は書かない。

**職歴**——逆時系列。役職ごとに：
**[役職]**、[会社] · [期間]
- [実績箇条書き：**行動動詞 → 何をしたか → 定量化されたインパクト**]。例：「オンボーディング離脱率を 18%→9% に削減し、約 $140k の ARR を解放。」
- 直近の役職は 3〜5 項目、古い役職は少なめに。職務ではなく実績を。

**スキル**——グループ化し、キーワード豊富に、求人票の言葉遣いを反映（ATS はここでマッチングする）。

**学歴**——学位、機関、年度、資格。

**調整メモ**（ユーザー向けに別添）：求人のどのキーワードを織り込んだか、カバーレターで埋めるべきギャップ。

## 品質チェック

- [ ] 職歴の各項目が**指標付きの実績**であり、職務（「〜を担当」）ではない
- [ ] 箇条書きは強い行動動詞で始まり、一人称代名詞を使わない
- [ ] 1 カラム、標準的な見出し、ATS 解析を壊す表・テキストボックス・図形なし
- [ ] ターゲット求人票のキーワードが自然に登場する（スキル欄 + 箇条書き）
- [ ] 長さがシニアリティに合っている（経験約 10 年未満は 1 ページ、最大 2 ページ）。最新・最関連が先頭
- [ ] 連絡先が完全で、サマリーがターゲット職種を名指ししている

## アンチパターン

- [ ] 職務を並べない——「チームを管理」は職責。「チームを 4 人から 11 人に拡大し、離職率を 30% 削減」が実績
- [ ] 複数カラム、表、ヘッダー/フッター、アイコンを使わない——ATS パーサーで崩れる
- [ ] 汎用履歴書を書かない——サマリー、スキル、力点をターゲット職種に合わせる
- [ ] ソフトスキルの埋め草（「勤勉なチームプレーヤー」）で水増ししない——結果で示す
- [ ] 指標を捏造・誇張しない——実数か、明確にそう示した妥当な見積もりを使う

## 基盤

実績主導・ATS 対応の履歴書実務（逆時系列、定量インパクトの箇条書き、キーワード整合）。
