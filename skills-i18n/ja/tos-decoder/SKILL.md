# 利用規約解読スキル

誰も規約を読まない——それがビジネスモデルだ。このスキルは代わりに読み、条項ごとに唯一重要な問いに答える：*本当に気にすべきか？* 利用規約の大半は防御的な定型文だ。価値は、定型ではない 3 つの条項を見つけることにある。

## このスキルが産出するもの

- 文書の順ではなく、現実世界への影響でランク付けされた発見
- 発見ごとの平易な「あなたが同意していること」と「気にすべきか？」の判定
- 結論：同意する / 目を開けて同意する / 避ける
- ひどい条項に対して実際にできること（設定、オプトアウト、代替サービス）

## 必要な入力

提供されていない場合のみ求める：

- **利用規約/プライバシーポリシーのテキスト**——全文または部分の貼り付け。抜粋の場合はある分を解読し、共有された範囲に欠けている高影響トピック（仲裁、データ共有、ライセンス、削除）を列挙する。
- **どんなサービスか**、どう使うつもりか（気軽な利用か業務利用か、オリジナル作品のアップロード、機微データの保存）。
- **特に心配していること**（具体的なものがあれば）。

## フレームワーク：深刻度スケール

現実の人間に何が起きるかで、最悪のものから順にランク付けする：

- 🔴 **お金や権利を実際に失わせ得る**——拘束力ある仲裁 + 集団訴訟の放棄（問題が起きたとき訴訟に参加できない）、第三者/データブローカーへの個人データの販売・共有、あなたのコンテンツへの広範な永続ライセンス（特に再許諾可能/AI 学習用）、「継続利用 = 同意」の一方的変更条項、支払い済み残高やコンテンツを没収するアカウント停止。
- 🟡 **異例——クリック前に知っておく**——実際には無効化にすぎない、またはバックアップを除外する「削除」、解約しづらい自動更新、アカウント閉鎖後のデータ保持、居住地から遠い裁判管轄、フィードバックは当社のものになる条項。
- 🟢 **標準的な定型文**——保証の否認、責任上限、利用ルール。名指しして、読者がそれらを心配しなくて済むようにする。

各 🔴/🟡 の発見に、*このユーザー*の申告した用途に合わせた一行の**「気にすべきか？」**判定を書く——例：「オリジナル作品をアップロードするなら Yes。眺めるだけなら無視してよい」。特に確認する：収集 vs 共有 vs 販売されるデータ。コンテンツライセンスの正確な範囲（永続か？再許諾可能か？削除後も存続するか？）。紛争はどう解決しなければならないか。規約はどう変更され得るか。削除は実際に何を削除するか。

## 出力形式

### 利用規約解読：[サービス名]

**1. 結論**——同意する / 目を開けて同意する / 避ける、を 2 文で。加えて単体で最悪の条項。

**2. 発見、影響順**

| # | あなたが同意していること（平易に） | 出典（引用行/条項番号） | 深刻度 | 気にすべきか？ |
|---|---|---|---|---|

**3. 削除の実態**——本文によれば、「アカウント/データを削除」は実際に何をするか。

**4. あなたにできること**——オプトアウト、設定、本文が提供していれば仲裁オプトアウトの期限、そして単純に受け入れるか去るかしかないもの。

成果物の末尾に、一字一句そのまま付す：*「これは平易な言葉での読解であり、法的・財務的助言ではありません——法律は法域により異なります。重要事項は資格を持つ専門家に確認してください。」*

## 品質チェック

- [ ] 発見は現実世界への影響でランク付けされ、文書自身の順ではない
- [ ] すべての 🔴/🟡 の発見が条項の実際の文言か条項番号を引用している
- [ ] すべての発見に、ユーザーの申告用途に結びついた「気にすべきか？」判定がある
- [ ] 標準的な定型文は明示的に 🟢 と表示——安心させることも成果物の一部
- [ ] 提供テキストにない高影響トピックは未確認として列挙し、問題なしと仮定しない
- [ ] 免責文が一字一句そのまま成果物に現れる

## アンチパターン

- [ ] 文書にない条項を創作しない——提供されたテキストだけを解読する
- [ ] バランスを装って危険信号を和らげない——「どこもやっている」は無害を意味しない
- [ ] 法域依存のルール（プライバシー権、仲裁の制限）を普遍として提示しない
- [ ] ありふれた定型文に憤慨を演じない——オオカミ少年は本物の発見を埋もれさせる
- [ ] 判定を省かない——「気にすべきか？」のない条項リストは、ただの短い利用規約だ

## 基盤

消費者契約レビュー実務——影響順の条項トリアージ、ライセンス範囲の読解、紛争条項の分析。
