# 服务条款解读技能

没人读条款——这本身就是商业模式。这个技能替你读，并对每个条款只回答那个真正重要的问题：*你到底该不该在意？*服务条款的大部分是防御性模板；价值在于找出那三条不是模板的。

## 这个技能产出什么

- 按现实影响排序的发现，而非按文档顺序
- 每项发现的大白话"你在同意什么"，附"我该在意吗？"结论
- 底线判断：接受 / 睁着眼接受 / 避开
- 对糟糕条款你实际能做什么（设置、退出选项、替代品）

## 所需输入

仅在未提供时索取：

- **服务条款/隐私政策文本**——完整或分段粘贴均可。若只有节选，解读现有内容，并列出所分享内容中缺失的高影响主题（仲裁、数据共享、授权、删除）。
- **这是什么服务**，以及他们会怎么用（随便用用还是商用、上传原创作品、存储敏感数据）。
- **他们最担心什么**（如有具体顾虑）。

## 框架：严重程度分级

按对真实的人会发生什么排序，最坏的在前：

- 🔴 **会让你损失金钱或权利**——强制仲裁 + 集体诉讼弃权（出事时你无法参与集体诉讼）、向第三方/数据经纪商出售或共享个人数据、对你内容的宽泛永久授权（尤其可再授权/用于 AI 训练的）、"继续使用即视为同意"的单方面修改条款、账号终止时没收已付余额或内容。
- 🟡 **不寻常——点击前要知道**——实为停用或不含备份的"删除"、难以取消的自动续费、销号后的数据保留、远离住地的管辖法院、"反馈归我们所有"条款。
- 🟢 **标准模板**——免责声明、责任上限、可接受使用规则；点名说明，让读者不必再为它们担心。

对每项 🔴/🟡 发现，写一行针对*该用户*声明用途的**"我该在意吗？"**结论——例如"如果你上传原创作品，要在意；只是围观就忽略。"专项检查：收集 vs 共享 vs 出售的数据；内容授权的确切范围（永久？可再授权？删除后仍有效？）；争议必须如何解决；条款如何修改；删除到底删了什么。

## 输出格式

### 服务条款解读：[服务名称]

**1. 底线**——接受 / 睁着眼接受 / 避开，两句话，外加最糟糕的那一条。

**2. 发现，按影响排序**

| # | 你在同意什么（大白话） | 出处（引用行/章节） | 严重程度 | 我该在意吗？ |
|---|---|---|---|---|

**3. 删除的真相**——按文本所写，"删除我的账号/数据"实际做了什么。

**4. 你能做什么**——退出选项、设置、文本提供的仲裁退出窗口，以及哪些纯属要么接受要么走人。

在产出物结尾原文附上：*"这是一份大白话解读，不构成法律/财务建议——法律因司法辖区而异；任何关键事项请向合格专业人士确认。"*

## 质量检查

- [ ] 发现按现实影响排序，而非按文档自身顺序
- [ ] 每项 🔴/🟡 发现均引用条款原文或章节号
- [ ] 每项发现都有与用户声明用途挂钩的"我该在意吗？"结论
- [ ] 标准模板明确标注 🟢——让人安心也是产品的一部分
- [ ] 所提供文本中缺失的高影响主题被列为未审查，而非假定没问题
- [ ] 免责声明逐字出现在产出物中

## 反模式

- [ ] 不要虚构文件中不存在的条款——只解读所提供的文本
- [ ] 不要为显得客观而软化警示信号——"大家都这么写"不等于无害
- [ ] 不要把依赖司法辖区的规则（隐私权利、仲裁限制）说成普适
- [ ] 不要对普通模板表演愤怒——狼来了会埋掉真正的发现
- [ ] 不要省略结论——没有"我该在意吗？"的条款清单只是一份更短的服务条款

## 依据

消费者合同审查实践——按影响分诊条款、授权范围解读、争议条款分析。
