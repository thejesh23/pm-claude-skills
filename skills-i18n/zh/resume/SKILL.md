# 简历技能

一份简历只有约 7 秒钟的人工浏览时间，以及一次 ATS 扫描。所以它必须*易于扫读*、*以成就为主线*、*关键词对齐*——而不是职位描述的复述。这个技能把你的经历转化为量化的成就要点，采用单栏结构（ATS 安全），并针对目标职位定制。可用 **Paper** 或 **Modern** PDF 主题导出，获得排版效果。

## 从粗略素材开始工作

你经常只会拿到零散笔记、不完整的经历，或者仅仅一个目标职位。**无论如何都要交付一份完整、可直接使用的简历**——不要停下来提问，也不要留下 `[公司名]`、`[补充数据]` 这类占位符。缺少细节时，根据素材和目标职位推断一个具体、合理的内容，并把所有推断内容标注为*（推测——请确认）*，让用户知道需要核实。一个具体且有标注的推测，永远好过一个空白或一个追问。

每项成就都要量化。**优先使用用户提供的真实数字**——用户给出的数据要原样使用。只有当指标确实缺失时，才把职责转写为以结果为导向的成就（写*结果*，例如"上线移动应用 v1，并获得第一批用户"）；只有在能给出**站得住脚的保守估计**时才补充数字，并标注*（推测——请确认）*。绝不能在真实的简历上悄悄编造或夸大数字——一个未标注的虚构指标，比缺失指标更糟糕。

只输出**成品简历**（及其简短的定制说明）——不要开场白，不要"这是您的简历"，不要对自己做了什么发表元评论。

## 输入（未提供的可推断——标注推测）

- **目标职位 / 职位描述**——让简历得以定制和关键词对齐（通用简历必败）。
- **你的经历**——职位、时间和所做/所成之事（零散笔记也可以；技能会负责量化）。
- **技能、工具、教育背景、证书。**
- **资历与格式偏好**——倒序时间（默认）或职能式；一页（多数情况）或两页。

## 输出格式

单栏、ATS 友好的简历，按以下顺序：

### [姓名]
[目标职位] · [城市 / 远程] · [邮箱] · [电话] · [LinkedIn/作品集]

**概要**——2–3 行：你是谁、你最有力的证明、你的目标方向。不要"结果导向的专业人士"之类的空话。

**工作经历**——倒序时间。每个职位：
**[职位]**，[公司] · [起止时间]
- [成就要点：**动作动词 → 做了什么 → 量化影响**]。例如"将新用户流失率从 18% 降至 9%，带来约 $140k ARR。"
- 近期职位 3–5 条要点，早期职位更少。写成就，不写职责。

**技能**——分组呈现、关键词丰富，呼应职位描述的用语（ATS 靠它们匹配）。

**教育背景**——学位、院校、年份；证书。

**定制说明**（单独给用户）：织入了职位描述中的哪些关键词，以及需要在求职信中弥补的差距。

## 质量检查

- [ ] 每条经历要点都是**带指标的成就**，而非职责（"负责……"）
- [ ] 要点以有力的动作动词开头；不用第一人称代词
- [ ] 单栏、标准标题，无表格/文本框/图形等破坏 ATS 解析的元素
- [ ] 目标职位描述中的关键词自然出现（技能区 + 要点中）
- [ ] 篇幅匹配资历（约 10 年以下 1 页；最多 2 页）；最新、最相关的在前
- [ ] 联系信息完整，概要中点明目标职位

## 反模式

- [ ] 不要罗列职责——"管理团队"是职责；"团队从 4 人扩至 11 人，流失率降低 30%"才是成就
- [ ] 不要用多栏布局、表格、页眉页脚或图标——它们会在 ATS 解析器中乱码
- [ ] 不要写通用简历——概要、技能和侧重点都要针对目标职位定制
- [ ] 不要用软技能空话填充（"勤奋的团队合作者"）——用结果来证明
- [ ] 不要编造或夸大指标——用真实数字，或明确标注的合理估计

## 依据

以成就为主线、ATS 感知的简历实践（倒序时间、量化影响要点、关键词对齐）。
