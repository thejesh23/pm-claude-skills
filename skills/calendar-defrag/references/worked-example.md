# Worked Example — Calendar Defrag

## The brief

Marcus, a data-platform EM with 8 reports, wants deep-work time for a migration design doc due in two weeks. Peak hours: mornings. Untouchables: Monday staff meeting, the exec review, all 1:1s. The agent may move only meetings he owns; others get drafted proposals.

## The output

# Calendar Defrag: week of Nov 3

**The bill:** 23 meeting-hours (× attendees = **74 person-hours**) · fragments waste **6.5h** (nine sub-45-min gaps) · largest focus block currently possible: **50 minutes**. Six-month bill of the recurring set: **~$310k in person-hours** at loaded cost.

| Meeting | Cost | Verdict | Move |
|---|---|---|---|
| Data sync (weekly, 60m, 9 people) | 9h/wk → 234h/6mo | **Halve + agenda** | 30m, decisions-only; status moves to the thread (owned by Marcus — direct move) |
| "Pipeline check-in" (2×/wk, 30m, 4p) | 4h/wk | **Make async** | Dashboard + Friday thread; draft note to attendees prepared |
| Vendor demo (Thu 10:00, 60m) | 1h | **Move** | → Thu 15:00; Marcus owns the invite; morning freed |
| Architecture review (Wed 11:00, 90m) | 6h | **Keep, move edge** | → Wed 14:00 — owned by Priya: *proposal draft prepared, not moved* |
| 1:1s ×8 (30m) | 4h | **Keep (untouchable)** | Clustered proposal: Tue/Thu afternoons — 8 individual drafts prepared, each opt-in |
| Staff meeting, exec review | — | Untouchable | — |

**After defrag:** three 2h+ morning focus blocks (Mon, Wed, Thu 9:00–11:15, labeled *"Migration design — booked by calendar-defrag, decline-with-comment welcome"*), **7.5h/week returned**, fragment waste 6.5h → 2h.

**Action plan (approved before execution):** create 3 labeled blocks · move 2 Marcus-owned events · halve 1 Marcus-owned event with the new agenda in the description · save 11 draft proposals (never sent) for others' meetings.

*[Executed. Verification: week re-read — 3 blocks present at approved times, 2 events at new times, sync at 30m with agenda, 11 drafts listed by recipient; before/after fragment waste: 6.5h → 2h. One conflict flagged: the moved vendor demo now abuts Marcus's 15:30 1:1 — left standing per approval, noted for next week.]*

## Why it's shaped this way

- **The bill opens in person-hours and six-month dollars** — a $310k number changes meeting culture; "you have a lot of meetings" changes nothing.
- **Verdicts follow the preference order** (async > halve > move-to-edge) and every removal/change affecting others is a *draft proposal*, never a unilateral move — fragmentation exported is not fragmentation solved.
- **Focus blocks were placed first, on stated peak hours, and labeled** — an unlabeled empty slot gets colonized in a day, per the anti-patterns.
- **Untouchables were respected even where they're the problem** (8×30m 1:1s fragment the week; the fix offered is clustering, opt-in per person).
- **Verification prints before/after numbers and one honest wrinkle** — the abutting-meeting note is what real execution reports look like.
